=========================================
16. Forms and Validation
=========================================

📌 What is a Form?

A Form is used to collect user input such as:

• Name
• Email
• Password
• Phone Number
• Address
• Feedback

HTML uses the <form> element to create forms.

=========================================

📌 What is Validation?

Validation is the process of checking whether the user input is correct before submitting the form.

Validation helps prevent invalid or incomplete data.

=========================================

📌 Types of Validation

1. HTML Validation
2. JavaScript Validation

=========================================

📌 HTML Validation

HTML provides built-in validation using attributes like:

• required
• minlength
• maxlength
• pattern
• type="email"

Example:

<input type="email" required>

=========================================

📌 JavaScript Validation

JavaScript allows custom validation based on application requirements.

Example:

if(name.value === ""){
    alert("Please Enter Name");
}

=========================================

📌 Form Events

• submit
• input
• change
• focus
• blur

Most commonly used event:

submit

=========================================

📌 preventDefault()

preventDefault() stops the default form submission and page reload.

Syntax:

event.preventDefault();

=========================================

📌 Validation Example

Check Empty Name

if(name.value === ""){
    alert("Please Enter Name");
}

Check Empty Email

if(email.value === ""){
    alert("Please Enter Email");
}

Check Password Length

if(password.value.length < 6){
    alert("Password Must Be At Least 6 Characters");
}

=========================================

📌 Interview Questions

1. What is a Form?
→ Used to collect user input.

2. What is Validation?
→ Checking user input before submitting.

3. Why is Validation important?
→ To ensure correct and secure data.

4. Which event is commonly used for forms?
→ submit

5. What does preventDefault() do?
→ Stops the default form submission.

6. Difference between HTML Validation and JavaScript Validation?
→ HTML provides built-in validation, while JavaScript provides custom validation.

=========================================

📌 Best Practices

✔ Always validate user input.
✔ Use meaningful input names.
✔ Display user-friendly error messages.
✔ Use preventDefault() with JavaScript validation.
✔ Never rely only on client-side validation.

=========================================

📌 Common Mistakes

❌ Forgetting event.preventDefault().
❌ Not checking empty fields.
❌ Using incorrect input types.
❌ Ignoring password length validation.
❌ Depending only on HTML validation.
